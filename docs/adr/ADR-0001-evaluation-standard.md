# ADR-0001 — 評価基準：「勝てる日の選定器 × 戦略」を、エントリー時リターンで評価する

- Date: 2026-07-13
- Status: **ACCEPTED**（owner 決定）
- Scope: **`_stocktrading` / `_bellwether` / `_sacrificial-lamb` / `_blacksheep` の4リポジトリ共通**
- Supersedes: 「毎日・全銘柄に機械的にブロードキャストする戦略を、日次/年次の集計 PnL で評価する」という
  暗黙の前提（過去の全 family がこの前提の下で設計され、そして死んだ）

---

## 1. 決定

戦略は今後、**単体では評価しない**。評価単位は次の**組**とする:

> **(G, S) = 「勝てる日を選定する仕組み G」 × 「その日に執行する戦略 S」**

そして **G が発火してエントリーした時のリターン**で評価する。

- **毎日 結果を出す必要はない。** G が発火しない日に何もしないことは、失点ではない。
- **主評価指標 = net return per entry**（エントリー1回あたりの、摩擦控除後リターン）。
- 日次平均・年次集計 PnL は**ランキングに使わない**（§4 の理由）。報告はするが判定には使わない。

---

## 2. なぜこう変えるか

`_bellwether` の摩擦統一理論（`why-sl-rejected-v19-survived-mgmt-artifact-2026-06-04.md`）が既に示している:

> 生存は **`1トレードのグロス ÷ (脚数 × スプレッド越え)`** で決まる。signal 強度は red herring。

死体は全て「固定ルールを母集団にブロードキャストし、毎日スプレッドを跨ぐ」形だった。
分母（越えた回数）を膨らませ、分子（1回のグロス）を薄める設計そのものが死因である。

**「稀に・確信の時だけ」は分母を下げる唯一の操作**であり、それを正しく採点するには、
**「エントリーしなかった日」を分母から外した指標**が要る。日次平均は選択性を希釈して罰する。

---

## 3. 正式な定義

対象期間の各日 `d` について:

- **G(d) ∈ {0,1}**（および銘柄選択）: **`d` のエントリー時刻より前の情報のみ**で決定される。
- `E` = G が発火した全エントリーの集合。`n = |E|`、`D` = 発火日数。
- 各エントリー `e` について `net(e) = gross(e) − friction(e)`（bps）。

**主指標**

| 指標 | 定義 |
|---|---|
| `net_per_entry` | `mean_{e∈E} net(e)` |
| `ratio` | `mean gross / mean friction`（= 摩擦の何倍か） |
| `t_net` | `net(e)` の**日クラスタ** t 値（クラスタ = 営業日。銘柄は日内で共変するため） |
| `hit_rate` | `net(e) > 0` の割合 |

**必ず併記する（判定には使わないが、隠すと嘘になる）**

- `n`（エントリー数）、`D`（発火日数）、`D / 全営業日`（発火率）
- `net_per_entry × n / 年`（＝実際に取れる年間 bps。これが owner の最終目的）
- 左裾: worst trade、`p1`、MAE 中央値・p10

---

## 4. ★ガードレール（これが無いと本指標は best-cell マイニング機械になる）

CLAUDE.md は既にこう警告している:

> **net PnL でランキングすると「取引しないこと」が報われる。**

**本 ADR の指標には、これと対称の罠がある:**

> **net per entry でランキングすると「一度しか取引しないこと」が報われる。**

選定器 G は「**勝てる日を選ぶ**」ことも「**勝った日を後から指す**」ことも、数字の上では同じに見える。
2026-07-13 セッションの実例（`docs/analysis/event-initiation-tail-verdict-2026-07-13.md` §6）:

> 「−7〜−10% の下落銘柄を空売り」は **エントリー当たり +79.5bps**（ratio 4.2）に見えた。
> しかし利益の実体は **2024-08-05 の一日**（円キャリー巻き戻し）だけ。
> その日を除くと **−23.2bps / net −42.2bps**。**選定器が優秀なのではなく、暴落が一度あっただけ。**

したがって **(G, S) は以下を全て満たして初めて採点対象**とする。1つでも落ちたら KILL。

### G1. 因果性（lookahead 禁止）
G は**エントリー時刻より前**の情報のみを使う。当日分位・当日正規化の閾値は禁止（lesson#26。
#67 は +6.66bps/t=5.38 の PROMISING が因果化して −1.7bps に反転した）。

### G2. マッチドヌル — **選定器そのものを検定する**
**同じ発火回数 `D` を、ランダムな日に割り当てた**ときの `net_per_entry` と比較する。

> **G の価値 = (選定日の net/entry) − (ランダム日 同数の net/entry)**

これが有意に正でなければ、**G は何も選んでいない**（S が良いだけ、あるいは期間の符号を拾っただけ）。
同様に「同日・同サイド・ランダム銘柄」ヌルで**銘柄選択**も検定する。

### G3. 集中度（一発屋の禁止）
- **単一日**の PnL シェア **> 0.30 → KILL**（← 2024-08-05 型を直接殺す）
- **単一銘柄**の PnL シェア **> 0.30 → KILL**
- 上位3日を除いても符号が保たれること。

### G4. 最小発火数
`n ≥ 30` かつ `D ≥ 20`。これ未満は**評価不能**（PROMISING とすら呼ばない）。
「年に2回しか撃たない」は戦略ではなく逸話である。

### G5. IS / OOS
G と S のパラメータは **IS で凍結**し、**OOS では一切触らずに適用**する。
OOS で符号反転 → KILL。**IS 側だけで成立する形は無価値**（今セッションの top5 フェードは
IS で net マイナス、OOS だけプラスだった）。

### G6. 摩擦の下限（既存ルールを継承）
`ratio = gross / friction ≥ 3.0`。摩擦は JPX 呼値ラダー由来の実測値を使い、
stress（4tick + 10bps）でも生存を確認する。

### G7. 執行可能性
**「そこで約定できたか」を必ず問う。** 今セッションの最大の教訓:

> 極端ムーバー・ショートは ratio 6.4 だったが、**利益は値幅上限への近さに単調増加**していた
> （張り付き銘柄で +174bps）。**儲かるスライスほど売り板が無い。**

- 空売りなら: 貸借銘柄か / 新規売建在庫があるか / ストップ高に張り付いていないか
- 記録された始値が「**実際には当てられない気配値**」でないか
- 価格グリッド（呼値・値幅制限ラダー）の副産物でないか（**今セッションはこれで1セル死んだ**）

### G8. honest-N
**選定器を探すこと自体が探索である。** G の変種を試すたびに N はインクリメントされる。
「戦略は同じで選定器だけ変えた」は**新しい family として数える**。N はリセットしない。

## 5. 採点フォーマット（全リポジトリ共通）

```
(G, S) = <selector名> × <strategy名>
  発火    : D=___日 / 518日 (___%)   n=___ entries   (___ /発火日)
  主指標  : net/entry = ___ bps   ratio = ___x   t_net = ___   hit = ___%
  ヌル    : ランダム同数日 = ___ bps  →  GAP = ___ bps (t=___)     [G2]
  集中度  : 最大日 ___%  最大銘柄 ___%  上位3日除外後 = ___ bps    [G3]
  IS/OOS  : IS = ___ bps   OOS = ___ bps                          [G5]
  左裾    : worst = ___%   p1 = ___%   MAE中央値 = ___ bps
  執行    : <借株/張り付き/グリッド> の検証結果                    [G7]
  年間    : net/entry × n/年 = ___ bps/年
  VERDICT : PASS / KILL(<落ちたゲート>)
```

---

## 6. 帰結

- **「毎日撃つ」設計は今後 提案しない。** 分母を膨らませるだけで、摩擦統一理論が既に否定している。
- **既存の棄却判定は覆らない。** 過去の family は「gross < friction」または「gross が触れない場所にある」
  で死んでおり、これは分母の数え方を変えても復活しない。**本 ADR は蘇生装置ではない。**
- **本 ADR が新たに可能にするのは、「発火率が低いこと」を減点せずに評価すること**だけである。
  低発火率は、G2〜G4 を通過して初めて美点になる。通過しなければ、それはただの過少サンプルである。
