## 0. 一行の結論

> **シグナルは本物で 24 年間符号安定（gross ~6bps, t=16.2）。だが gross が摩擦の 3 倍に届く
> 執行可能ホライズンが存在せず、取れる部分は「寄り板寄せの約定プリント」＝実際には当てられない
> fill に 100% 集中している。→ 戦略としては KILL（G6 摩擦・G7 執行可能性）。シグナル自体は棄却しない。**

README が自ら疑っていた「固定 09:00 アンカーの計測アーティファクト」は正しかった。実約定タイミングで
測り直すと **net は負**、というのが全パターンで再現した。

## 1. 検証したデータと網羅範囲

| データ | 範囲 | 用途 |
|---|---|---|
| 分足 `stocks_minute` | 2024-01 .. 2026-02（**509 営業日**, 18 銘柄） | 主検定（イントラデイ出口） |
| 日足 `stocks_daily` | 2001 .. 2026（**6,159 営業日**） | 24 年レジーム頑健性（open→close） |
| 歩み値 `stocks_trades` | 2024-01 .. 2024-08（tick） | 分足ハーネスの忠実性検証 |
| 板 `stocks_board_snapshot` | 2026-07-07..09（3 日・**寄り前サイクル 0 件**） | ← **寄り前指標板の履歴は事実上存在しない** |

> ⚠️ **重要**: README の凍結シグナル入力（寄り前 08:59:30 の指標板 mid）は、履歴が 3 日しか無く、
> しかもその 3 日とも寄り前サイクルを含まない。**note 仕様そのままの signal はバックテスト不能。**
> 代わりに「寄り（始値）の gap 差」を signal に使った（寄成で入る以上、これが実際に取引する realized 版で、
> かつ寄り前指標 mid とはほぼ一致する）。この置換により **509 日 × 全パターン**の網羅検定が可能になった。

**掃引したパターン（実装コスト度外視）**: 方向（fade / momentum）× 脚構造（2脚 / 片脚short / 片脚long /
ベータ）× 入口ラグ（0/30/60/120s）× 出口（30s〜3600s, from-fill/固定）× 選別（絶対閾値 0〜800bps /
ペア相対 z 1.0〜3.0）× ペア集合（凍結15 / 全同業ペア28 / クロス業種ヌル）× IS/OOS × 摩擦（1tick/2tick+stress）。

## 2. (G,S) 採点（ADR-0001 フォーマット）

### 本命＝仕様通り: 凍結15 × 毎日 × 2脚フェード × 180秒
```
発火    : D=509日   n=3,925 entries
主指標  : net/entry = -3.51 bps   gross=3.71  fric=7.22  ratio=0.51   t_net=-5.16   hit=45.8%
IS/OOS  : IS=-3.39   OOS=-4.57（両側マイナス）
執行    : gross の 100% が寄り約定プリントに集中（§3）
VERDICT : KILL（G6 ratio<3、net<0、両側 OOS 負）
```

### 選別で最も良く見えたセル: z>2.0 × 2脚フェード × 180秒
```
発火    : D=64日   n=74
主指標  : net/entry = +10.23 bps   gross=18.02  fric=7.79  ratio=2.31   t_net=+1.57   hit=60.8%
ヌル    : ランダム同数日 = -2.41 bps → GAP=+12.64（G2）
集中度  : 最大日 0.37   最大ペア 0.31（G3 は通る）
VERDICT : KILL（G6 ratio 2.31<3.0 / t=1.57 有意でない / 数十セル走査後の 1 つ＝多重比較）
```

### 片脚 long（出遅れ買い）× thr>200 × 180秒（friction 半減狙い）
```
発火    : D=125日   n=192
主指標  : net/entry = +11.92 bps   gross=19.01  fric=7.10  ratio=2.68   t_net=+1.44   hit=52.1%
VERDICT : KILL（ratio 2.68<3、有意でない。かつ §3 の通りこれは「出遅れ株が寄りから戻る」
          単一銘柄効果で、ペア構造の寄与ではない）
```

**ratio ≥ 3.0 に到達したセルは全掃引で 0 個。** 最良は 2.68、しかも統計的に有意でない。
2tick+2bps ストレスを掛けると最良セルも ratio 0.31 に崩壊する。

## 3. なぜ死ぬか — 3 つの決定的診断

**[C] エッジは寄り約定プリントに 100% 集中し、60 秒で消える（G7・致命）**

| 入口 | 出口 | gross(片脚long) | net | t |
|---|---|---|---|---|
| 寄り(0s) | +180s | **6.47** | -0.80 | -1.21 |
| 寄り+60s | +180s | **0.15** | -7.16 | -4.16 |

寄りの板寄せ価格でちょうど約定できて初めて取れる。60 秒待てば消える。これは README の kill#5
（遅延寄り・特別気配で寄成が本当に約定するか）そのもの。**完全 fill を仮定してすら net ≈ 0〜負。**

**[A] 「ペア」はほぼ無意味 — 正体は単一銘柄の寄り gap リバージョン（G2）**

全 18 銘柄・全日で無条件に「寄り買い→+180/600s」を測ると、**gap が低い脚（＝出遅れ）だけが
+3.6/+6.6bps 戻り、gap が高い脚は ~0**。ペアを組まなくても同じ効果が全銘柄に出る（無条件 t=0.68〜1.14
で**有意ですらない**）。フェードの短脚（リーダー空売り）は gross **+0.95** とほぼ無エッジ。
→ 「同業ペアの乖離回帰」ではなく「出遅れ株が寄りから少し戻る」普遍的マイクロ構造で、しかも弱い。

**[歩み値] ハーネスは忠実（自作の分足近似が捏造していない）**

tick 実データ（2024-01..08, n=1,679）で片脚long 寄り→+180s = gross **4.96bps / t=1.96**。
分足ハーネスの ~6.5bps と同符号・同オーダー（歩み値は寄り直後の初約定を使うため一段低いのは [C] と整合）。

## 4. ただしシグナル自体は本物（棄却しない）

日足 24 年（open→close フェード, gross bps, 摩擦前）:

| 期間 | n | gross | t |
|---|---|---|---|
| 2001-2005 | 11,271 | 6.29 | 6.14 |
| 2006-2010 | 12,238 | 6.61 | 7.24 |
| 2011-2015 | 14,163 | 6.67 | 8.82 |
| 2016-2020 | 17,005 | 5.90 | 8.16 |
| 2021-2026 | 17,929 | 5.72 | 6.89 |
| **全期間** | **72,606** | **6.19** | **16.20** |

同業ペアの相対乖離は 24 年どのレジームでも +6bps 前後で戻る（gap>150 で ~15bps, t~10）。
**これは統計的に紛れもない。**問題は「本物か」ではなく「摩擦を、当てられる fill で 3 倍超えるか」で、
その答えが No というだけ。~6bps は 2脚の呼値越え摩擦 ~7bps を超えられず、~15bps に増やすには
gap>150 かつ引けまで保有（＝全日分散を被る）が要り、それでも ratio ~2.2。

## 5. 結論と ADR-0001 上の位置づけ

- **戦略 = KILL。** 落ちたゲート: **G6（ratio<3 が全セル）/ G7（執行可能な唯一のスライス＝寄りプリントが
  取れない）**。補助的に IS 負・多重比較・単一銘柄効果への還元（G2）。
- **シグナル = 棄却しない。** 同業ペア相対リバージョンは実在し 24 年安定。ただし retail の taker 執行では
  摩擦に負ける。復活の唯一の筋は **maker 執行（寄りの板に受動的に置いて板寄せに巻き込まれる）**だが、
  それは別 family（#66 の maker 軸）の設計問題であり、本 taker ハーネスの延命ではない。
- **live/kabu_pair_open_fade.py は arm しない。** フォワードで測っても net は負に出る公算が高く、
  それは既にオフラインで 509 日 × 全パターンで判明済み。デモを回す価値は「寄成の実現率（kill#5）を
  実測する」点にのみ残るが、判定（採用/棄却）はこの文書で確定。
