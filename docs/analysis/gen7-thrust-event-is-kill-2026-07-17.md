## 起案の経緯 — 6世代の KILL に反する owner 方針から

owner 方針（2026-07-16 夜）: 「ML-Agent で足読みスキャル（高回転）にこだわる。数円の
小さな利益を何百回と繰り返してコツコツ利益を積み上げる」。これは gen1b/2/3/4/5/6 の
測定と正面から矛盾するため、実装前に grill-me で対立点を解消した（2026-07-17）。

grill-me の結果、owner は反論として**実トレードの存在証明**を提示した:
`_sacrificial-lamb` issue#1 — **9107 川崎汽船 2022-10-26**、40,000株を ~2381 で新規売り →
~2374 で返済買い、**gross 6.6円 = 27.7bps を2分で**取得（+¥264,000）。

これは handoff の反論（「3,000円株の数円 = 2円 = 6.7bps」）が**過小に見積もっていた**。
実物は 4 倍であり、gen4 の 2.46bps / gen5 の 1.54bps の **11〜18 倍**、friction 14bps の
**約2倍**。owner が指す現象は gen2-gen6 が測った領域とは別であり、(A) として筋が通る。

owner 承認の設計（grill-me で確定）:
- 検出器は issue#1 から**凍結移植**。掃引しない。マッチは選び好みせず**全件抽出・全件測定**。
- ユニバース = 「**最高値更新した株などイナゴが寄ってくる人気株**」。3 定義を並べる。
- short（down-thrust）と long（up-thrust 鏡像）を**同時に**測る（G7 空売り制約の退路）。

## 起案根拠 — 前世代がイナゴ株を構造的に排除していた（新発見）

gen5/gen6 は「gross の源泉の当てなしに新 family を起案しない」と定めた。本 family には
初めて具体的な当てがある。2026-07-17 の調査で判明した 3 点:

1. **前世代のユニバースはイナゴ株の排除装置だった。** `xsec/universe.py` は
   「前月末までの trailing 60 日 **median** TurnoverValue 上位 400・**月次**リバランス」。
   median は定義上スパイクを無視する推定量であり、イナゴ株はスパイクで定義される銘柄。
   さらに月次リバランスは月中の発火を翌月まで取りこぼす（入る頃には祭りが終わっている）。
2. **ADR `2026-07-16-end-bar-only-testa-scalping` の自己診断と整合する。** 同 ADR は
   「メガキャップの公開 OHLCV は裁定済みの集計情報」と述べる。gen2/gen3 = メガキャップ
   17 銘柄、gen4/gen5/gen6 = 代金上位 400。**6世代すべて裁定済みの銘柄群だけを測っていた。**
3. **owner の実トレード銘柄 9107 は、全世代のユニバースに一度も入っていなかった。**
   しかも経済的理由ではなく**バグ**による（次節）。

したがって本抽出は、同 ADR の「根拠が崩れる条件」1（未使用の独立期間・事前登録済み評価・
十分な取引数・コスト控除前後）の**事前登録実行**として位置づける。勝手な再開ではない。

## 派生発見 — gen4/gen5 のユニバースに data-coverage バグ

`scripts/gen4_xsec_pipeline.py:_has_minute()` は
`MINUTE_COVERAGE_REQ = ("2024-04-01", "2025-04-30")` を要求する。分足が**封印中の OOS 月
(2025-04) まで伸びている銘柄だけ**が通る。

- **9107 の分足は 2025-03-31 で終わる** → 除外。売買代金 **44 位**（24.6B円。9101=25.2B が
  22位、9104=25.5B が 28位で採用）。経済的理由ではなくデータ都合。
- 9107 だけではない。**代金上位 400 のうち 53 銘柄 (13.2%)** が同じ理由で落ち、
  401〜453 位に静かに差し替わっていた。落ちたのは 39, 43, 44, 64, 116 位…と上位を含む。

**この バグは gen4/gen5 の KILL を覆さない。** gross 2.5bps vs friction 14bps という
6〜20 倍の差は、ユニバースの 13% 入れ替えでは埋まらない。覆すのは「owner の実トレードの
銘柄は検証済みである」という暗黙の前提のみ。gen7 はこのバグを継承していない
（カバレッジ要件を抽出窓 2024-04-01〜2025-03-31 に合わせ、9107 と他 52 銘柄が復帰）。

## 設計

**検出器（issue#1 から凍結移植・掃引禁止）** — `down_thrust_scalp_01._down_thrust_metrics`:

    cum_ret = (close[i] - close[i-3]) / close[i-3]
    vol_med = median(volume[i-19 .. i])
    down: cum_ret <= -0.005  かつ  volume[i] >= 3.0 * vol_med
    up  : cum_ret >= +0.005  かつ  volume[i] >= 3.0 * vol_med   （鏡像・新パラメータ 0）

**ユニバース（PIT 日次・事前登録）** — 前世代の median×月次の逆を行く（スパイク×日次）:

| 定義 | 内容 | D | stock-days |
|---|---|---|---|
| U1 | 新高値(60日・**狭義**更新) AND 代金急増(>=3x 20日median) | 194 | 564 |
| U2 | 代金急増のみ | 244 | 2,301 |
| U3 | 新高値のみ | 243 | 6,194 |

hard constraint 全定義共通: プライム / median close >= 200 / median turnover >= 10億円。
**G7 執行可能性の proxy**: listed_info に貸借区分の列が無く履歴の貸借銘柄判定ができない。
「プライム かつ 平常代金 >= 10億円」を代替とした（9107 の 24.6B は通る）。これは proxy で
あり、PASS した場合のみ kabu API で実在庫を確認する予定だった。

**保守側への倒し方**（CLAUDE.md「fill が不確実な箇所は戦略に不利な側へ倒す」）:
- entry = 発火 bar の**次 bar の close**（発火 bar では約定しない = 因果）
- TP 到達は **close** 基準（髭で届いても取らない = 届きにくい側）
- stop 到達は **髭**（short=high / long=low）基準（引っかかりやすい側）
- 同一 bar で TP と stop が両方成立 → **stop 優先**（bar 内順序不明なので不利側）
- 日跨ぎなし（セッション境界リセット）。経路が最終 bar を超える場合は打ち切り

**分割**: TRAIN 2024-04-01〜2024-11-29 / VAL 2024-12-02〜2025-03-31。
**封印維持**: 2025-04（gen4/5/6 OOS）、2025-10〜2026-02（gen2/3 OOS）。

**friction**: gen4 の呼値ラダーモデル継承 `max(tick/price×1e4, 2.0) × 1.25 + 1.0`（往復）。

### G2 マッチドヌル — 検出器は「負の選定価値」を持つ

同一 stock-day・同一発火本数をランダム bar に割り当てた対照（seed 20260717）:

| side | thrust gross | random gross | GAP | z |
|---|---|---|---|---|
| short | +0.37 | +5.24 | **-4.87** | **-3.89** |
| long | +0.26 | +5.61 | **-5.35** | **-3.77** |

摩擦ゼロ期待値も thrust -2.64 vs random -0.48（short）、thrust -3.36 vs random -1.44（long）。

**thrust に乗ることは、同じ日の同じ銘柄のランダムな瞬間に入るより、有意に悪い。**
gen4/gen5 は「信号は実在するが小さい」（ヌル gap **+2.5bps / z=+5.0**）だった。gen7 は
**gap が負**（z≈-3.8）— 信号が無いのではなく、**逆選択**である。

解釈: 出来高スパイクを伴う急変後に順張りで入るのは、既に価格に織り込まれた情報に対して
**最後に乗る**行為である。owner の言葉を借りれば、**thrust で発火する者がイナゴ側**になる。

## 結論

**2022-10-26 は典型ではなく、当たりくじだった。**

owner の実トレードは TP +28bps 到達・MAE -21bps 耐久で勝った。イベント母集団では
その分岐は **~33%** で、**~61%** は先に stop に触れる。owner は **2:1 で不利なコイン**の
勝ち側を引いた。これは owner の腕を否定しない — **凍結検出器が owner の判断の代理として
粗すぎる**ことを意味する（下記「本結果が言っていないこと」）。

- **分足系 7 連続 KILL**（gen1b/2/3/4/5/6/7）。
- **イナゴ株ユニバース仮説は測定され、否定された。** 「前世代は裁定済み銘柄しか見ていない」
  は事実だったが、裁定されていない銘柄群でも分足 OHLCV に順張りエッジは無い。むしろ**負**。
- **maker/taker 両執行 × broadcast/event 両設計 × 裁定済み/イナゴ両ユニバース**で閉塞。
- ADR `2026-07-16-end-bar-only-testa-scalping` の「根拠が崩れる条件」1 は**満たされなかった**
  → **ADR の KILL 判定を維持**（見直し日 2026-10-16 は変更なし）。

## 本結果が言っていないこと（過大主張の禁止）

1. **owner の 2022-10-26 の判断が誤りだったとは言っていない。** 凍結検出器（3本 -0.5% +
   出来高 3倍）は、owner が実際に見ていたもの（板、テープ、地合い、海運セクターの文脈）の
   **粗い代理**にすぎない。本結果と最も整合する解釈は、**owner のエッジは分足 OHLCV に
   無い情報に由来する** — これは ADR の診断（板のミクロ構造こそが源泉）と同じ方向を指す。
2. **gen4/gen5 の KILL を本 family の結果が覆すことはない**（別 family・別測定）。
   逆に本 family の KILL も gen4/gen5 の universe バグとは独立（バグ修正後の測定である）。
3. **ヌル対照は時刻マッチしていない。** ランダム bar は日中一様、thrust は高ボラ時刻に
   偏る。この差は解釈に影響しうるが、**主判定（摩擦ゼロ期待値が全セル負）はヌルに依存しない**。
4. **G7 の貸借区分は proxy のまま。** 実在庫確認は PASS した場合のみ行う予定だったため未実施。
   KILL がそれ以前に出ているので、この未確認は判定に影響しない。
